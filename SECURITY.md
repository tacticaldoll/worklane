# Security Policy

## Supported Versions

Security fixes target the current `0.1.x` line before the project reaches 1.0.
Because the API is still pre-1.0, a security fix may require an additive API
change or a minor-version compatibility note.

## Reporting A Vulnerability

Please do not disclose exploit details in a public issue.

If GitHub Security Advisories are enabled for this repository, use the
repository's **Report a vulnerability** flow. Otherwise, open a minimal public
issue that says you have a security report and need a maintainer contact path,
without including reproduction details, credentials, payloads, or exploit code.

Useful private report details include:

- Affected crate and version.
- A short impact summary.
- Reproduction steps or proof of concept.
- Whether the issue requires a specific broker, lane name, payload shape, or
  deployment configuration.
- Any known mitigations.

The project will acknowledge valid reports, assess affected versions, and
coordinate a fix and advisory before public disclosure when possible.
